# Project Setup — College Facts Agent

One-time scaffolding instructions to lay out the repository. Work through the
checklist top to bottom. After this, the first real task is the CSV→Postgres
loader (see the end).

The project has three stages along language/data seams: **R does ETL → CSVs →
a loader lands them in Postgres → a Python agent queries them.**

## 4. Migration chores

- [ ] **git-lfs for the bundles.** `git lfs install`, confirm `.gitattributes`
      tracks `data/*.Rda`, then add the `.Rda` files. IPEDS is public domain, so
      committing is fine; lfs just keeps history clean.
- [ ] **Fix R source paths.** The modules assume the backbone is sourced first
      and reference it as `R/schools_pipeline.R`. Update any such reference to
      `etl/schools_pipeline.R`. Modules pull shared helpers (`get_table`,
      `ai_get`, `.out_path`) from the backbone, so it must be sourced before any
      module.
- [ ] **Guard the trailing auto-run lines.** `schools_pipeline.R` and each module
      end with a live call (e.g. `schools <- build_schools()`,
      `res_adm <- run_admissions_module()`) that is *not* commented out, so merely
      sourcing a file triggers a full build. Comment those lines out, or wrap them
      in `if (sys.nframe() == 0) { ... }`, so sourcing-for-helpers doesn't kick off
      a run.
- [ ] **Confirm the data layer.** Either the five `.Rda` files are present in
      `data/`, or run `Rscript etl/build_rda.R` to generate them (needs
      `mdbtools` + the pinned `jbryer/ipeds` package — see CLAUDE.md → "Data
      provenance").

## 6. First real task after setup

Write the loader — `load/schema.sql` + `load/load_to_postgres.py`:

- Four tables: `schools` and `value_labels` loaded as-is; all six `*_facts.csv`
  **unioned** into one long `facts` table (`unitid, year, metric, value,
  var_type`); all six `*_variables.csv` **stacked** into one `variables` catalog.
- Types/keys/indexes: `value` numeric, `year`/`unitid` integer; PK on
  `schools(unitid)` and `variables(metric)`; index on `facts(unitid, metric, year)`.
- **Integrity checks at load time:** every `metric` in `facts` must have a row in
  `variables`; every `unitid` in `facts` must exist in `schools`. Report orphans
  (this catches a value flowing through with no catalogued source/year/caveat).
- Idempotent: truncate-and-reload from the CSVs.

Then rework `agent/tools.py` to read the `variables` catalog instead of a
hardcoded field list. See CLAUDE.md → "Next steps" for the full sequence.
